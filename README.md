# Лабораторная работа №1 HTTP-методы и особенности Star Wars API (SWAPI) 

HTTP определяет набор методов запроса, которые задают семантику действия клиента по отношению к ресурсу. Метод `GET` используется для получения данных; он является безопасным (`safe`) и идемпотентным (`idempotent`), то есть не должен изменять состояние сервера, а повторение такого же запроса не меняет итоговый результат. Метод `HEAD` семантически аналогичен `GET`, но возвращает только заголовки без тела ответа; он тоже является safe и idempotent. Метод `OPTIONS` позволяет узнать, какие методы поддерживает ресурс, и также относится к safe и idempotent. Метод `POST` обычно применяется для создания нового ресурса или отправки данных на обработку; он не считается безопасным и обычно не является идемпотентным. Метод `PUT` используется для полной замены ресурса и является идемпотентным. Метод `PATCH` предназначен для частичного изменения ресурса и обычно не гарантирует идемпотентность. Метод `DELETE` служит для удаления ресурса и считается идемпотентным, так как повторный запрос не должен менять итоговое состояние после удаления.

На примере Star Wars API эти методы можно показать через адреса ресурсов. Базовый URL API — `https://swapi.py4e.com/api/`, а корневой ресурс возвращает ссылки на основные коллекции: `people`, `films`, `planets`, `species`, `starships`, `vehicles`. Реальный пример чтения данных: `GET https://swapi.py4e.com/api/people/1/` — запрос на получение персонажа Люка Скайуокера. Пример для получения только заголовков: `HEAD https://swapi.py4e.com/api/people/1/`. Пример для определения доступных методов: `OPTIONS https://swapi.py4e.com/api/people/`. При этом у SWAPI есть важная особенность: это открытый API без аутентификации, но в публичной версии он ограничен только получением данных. Поэтому запросы вида `POST https://swapi.py4e.com/api/people/`, `PUT https://swapi.py4e.com/api/people/1/`, `PATCH https://swapi.py4e.com/api/people/1/`, `DELETE https://swapi.py4e.com/api/people/1/` можно приводить как примеры общей HTTP-семантики, но не как реально поддерживаемые операции данного API.

Star Wars API удобен для изучения REST-подхода, потому что имеет простую и предсказуемую структуру. Во-первых, он поддерживает пагинацию: запрос к списку ресурсов, например `https://swapi.py4e.com/api/people/`, возвращает объект с полями `count`, `next`, `previous` и `results`, а переход между страницами выполняется через query-параметр `page`, например `https://swapi.py4e.com/api/people/?page=2`. Во-вторых, API поддерживает поиск через query-параметр `search`; пример: `https://swapi.py4e.com/api/people/?search=r2`. Поиск выполняется по частичному совпадению без учёта регистра. В-третьих, есть параметр `format`, например `https://swapi.py4e.com/api/planets/1/?format=wookiee`, который позволяет получить данные в альтернативном представлении. В-четвёртых, у каждого ресурса есть schema-эндпоинт вида `/api/<resource>/schema/`, например `https://swapi.py4e.com/api/people/schema/`, что позволяет посмотреть структуру ресурса и типы его полей. Кроме того, SWAPI использует ссылки на связанные сущности прямо в JSON-ответах: у персонажа в полях `homeworld`, `films`, `species`, `starships`, `vehicles` находятся URL других ресурсов, поэтому клиент может переходить по ним и получать связанные данные.

Таким образом, основные методы HTTP задают тип действия над ресурсом, а их свойства позволяют понять, изменяет ли запрос состояние сервера и что произойдёт при его повторении. На примере SWAPI удобно рассматривать как сами методы HTTP, так и практические особенности REST API: структуру URL, работу со списками ресурсов, пагинацию, query-параметры, поиск, схемы ресурсов и переходы по связанным ссылкам. При этом важно учитывать, что публичный Star Wars API предназначен в первую очередь для чтения данных, поэтому в реальной работе с ним используются главным образом `GET`, `HEAD` и `OPTIONS`, а остальные методы рассматриваются как теоретические примеры общей семантики HTTP.
